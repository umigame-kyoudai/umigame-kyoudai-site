# ブログ記事の書き方

記事は **1記事 = 1ファイル** で `content/blog/` フォルダに入っています。
ファイルを追加して main に push するだけで、自動デプロイされて公開されます。

## 新しい記事を公開する手順

1. `content/blog/` の既存記事（例: `miyakojima-drone-sup-guide.md`）をコピーする
2. ファイル名を新しい記事のURLにしたい英語スラッグに変える
   - 例: `miyakojima-fall-snorkeling.md` → 公開URLは `/blog/miyakojima-fall-snorkeling`
3. 先頭の `---` で囲まれた部分（フロントマター）を書き換える
4. その下に本文をMarkdownで書く
5. main に push（またはClaudeに「この内容で記事を作って」と依頼）

## フロントマターの項目

```yaml
---
title: "記事タイトル（検索結果に出る。32文字前後が目安）"
excerpt: "記事の要約（検索結果のdescriptionと記事冒頭の枠に表示される）"
author: "海亀兄弟編集部"
publishedAt: "2026-07-03"        # 公開日。並び順にも使われる
category: "アクティビティ"        # lib/data.ts の BLOG_CATEGORIES にあるものを使う
readTime: 8                       # 読了目安（分）
image: "/images/blog/xxx-v2.jpg"  # ヒーロー画像。public/images/blog/ に置く（1200x675推奨）
tags: ["タグ1", "タグ2"]
---
```

- 記事を後から更新した場合は `date: "2026-08-01"` を追記すると、sitemapの更新日に反映される
- タイトルや要約に `"` を使う場合は `\"` とエスケープする

## 本文で使える書式

- `## 見出し` / `### 小見出し`（`#` はタイトルと重複するため使わない）
- `**太字**`、`- 箇条書き`、`1. 番号リスト`
- リンク: `[表示テキスト](/plans/S1)` — プラン・予約・他記事への内部リンクを積極的に張る
- 表（`|` 区切り）は**使えない**。リストで代用する

## 記事の品質ルール

- 料金・年齢・時間などの事実は、プランページ・FAQと必ず一致させる
- 根拠のない数値（遭遇率◯%・口コミ◯件など）は書かない
- 記事末尾のCTA（予約ボタン枠）は `lib/blog/index.ts` の `getBlogPostCta` がタグから自動選択する
  - ドローン/空撮 → ドローンSUP、ナイトツアー/ヤシガニ → ナイトツアー、SUP → サンセットSUP、シュノーケル/子連れ → ウミガメ

## 公開後にやること

Google Search Console の「URL検査」で新記事のURLを検査し、
「インデックス登録をリクエスト」を押す（検索結果に載るのが早くなる）。
