# 海亀兄弟 匿名分析 → Google スプレッドシート

サイトから届く匿名イベントを固定列に保存し、日別集計・予約ファネル・プラン別売上・流入元別売上を自動表示します。

## 保存しない情報

- 氏名
- メールアドレス
- 電話番号
- 住所
- 予約時の自由記述
- IPアドレス
- Cookie
- ユーザーエージェント全文
- 訪問者・セッションを追跡する恒久ID

## 初回セットアップ

1. Google Apps Scriptで新しいプロジェクトを作成します。
2. `Code.gs` と `appsscript.json` の内容を同名ファイルへ貼り付けます。
3. `setupAnalyticsWorkbook` を1回実行し、Googleの権限を許可します。
4. 実行ログの戻り値にある `spreadsheetUrl` を開き、4シートが作成されたことを確認します。
5. `generateAnalyticsSharedSecret` を1回実行し、戻り値を安全な場所へ一時コピーします。
6. Apps Scriptの「デプロイ」→「新しいデプロイ」→「ウェブアプリ」を選びます。
7. 「次のユーザーとして実行」は自分、「アクセスできるユーザー」は全員にしてデプロイします。
8. 発行された `/exec` URLをコピーします。

## Vercel環境変数

海亀兄弟のVercelプロジェクトへ次を登録します。

| 名前 | 値 |
|---|---|
| `ANALYTICS_SHEETS_WEBHOOK_URL` | Apps Scriptのウェブアプリ `/exec` URL |
| `ANALYTICS_SHEETS_SHARED_SECRET` | `generateAnalyticsSharedSecret` の戻り値 |

Production / Preview / Development の必要な環境へ登録後、Productionを再デプロイします。秘密文字列をGitHub、チャット、スプレッドシートへ貼らないでください。

## 動作確認

1. 本番サイトを開きます。
2. ページを移動し、テスト予約を1件送信します。
3. `イベントデータ` に `page_view`、`booking_started`、`booking_submitted` が追加されることを確認します。
4. `booking_submitted` の行にプラン、人数内訳、金額があり、氏名・メール・電話番号がないことを確認します。
5. `ダッシュボード` と `日別分析` の数値が更新されることを確認します。

## シート構成

- `ダッシュボード`: 直近30日の主要指標、プラン別・流入元別集計、日別チャート
- `日別分析`: 日ごとのページ表示、予約開始、予約完了、予約失敗、参加人数、売上
- `イベントデータ`: 匿名イベントの固定列データ
- `設定・定義`: プライバシー方針とイベント定義

`イベントデータ` の列順はコードと集計式で使用しています。列の挿入・削除・並べ替えはしないでください。
