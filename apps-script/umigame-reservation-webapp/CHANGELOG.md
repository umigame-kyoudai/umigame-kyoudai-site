# 変更履歴

## 2026-07-18

### 重複予約の安全削除

- 予約詳細の一番下に「この予約を削除する」を追加
- 削除前に確認画面・予約番号入力・最終確認を必須化
- 単品予約は1行、セット予約は同じ予約番号の2〜3行をまとめて削除
- 削除前の全データを`削除済み予約`シートへ退避
- 同じ予約番号のGoogleカレンダー予定もまとめて削除
- LINE通知は送信しない仕様
- カレンダーまたは予約一覧の削除失敗時に自動復旧する処理を追加
- 成功・予約番号不一致・カレンダー失敗・シート失敗の模擬テストに合格

### セット予約の個別日付変更

- 昼夜セットのウミガメとナイトを別の日付・時間へ変更可能にした
- 海空セットは海亀とSUPを同日のまま、SUPを海亀開始90分後へ自動設定
- まるごと1日セットは海亀・SUPを同日連続、ナイトだけ別日へ変更可能にした
- 予約一覧・Googleカレンダー・確定・満席LINEを個別日付に対応
- 今日・明日・期間絞り込みで、セット内のいずれかの予定日を判定
- 個別日付の更新・カレンダー同期・失敗時ロールバック模擬テストに合格

### 確定前の予約日時変更

- 未対応予約に「予約日時を変更する」を追加
- 予約一覧とGoogleカレンダーの日付・時間を同時更新
- 更新失敗時にシートとカレンダーを元の日時へ戻すロールバックを追加
- 自動復旧にも失敗した場合は、両方の手動確認が必要と明示
- 海空セット・まるごと1日セットのSUPを海亀開始の90分後へ自動設定
- 昼夜セットの海亀・ナイト時間は個別入力
- 日時変更前に作成されたLINE送信待ちを自動取消し
- カレンダー認可用の `adminAuthorizeCalendarAccess` を追加
- 通常予約・昼夜セット・海空セット・まるごと1日セットの日時計算とカレンダーロールバック模擬テスに合格

## 2026-07-17

### 実際の運用順序へ調整

- 詳細画面で予約ステータスの確定を最初の操作として明示
- 開催場所の要確認を「明日の昼ツアー」と「本日のナイトツアー」に限定
- セット予約は昼予定のいずれかに場所があれば昼の案内済みと判定
- 担当スタッフを任意項目として表示

### 本番前の安全性・スマホ表示改善

- LINE送信にUUIDリトライキーを付与し、通信再試行時の二重送信を防止
- LINE送信成功後は履歴やU列の記録に失敗しても再送されない順序へ変更
- ステータス通知の送信済み情報をスクリプトプロパティにも保存
- 履歴記録失敗をLINE送信結果と分離し、管理画面へ警告表示
- スマホの確定売上カードで長い金額が切れないよう調整
