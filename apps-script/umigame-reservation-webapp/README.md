# 海亀兄弟 予約管理Webアプリ

Googleスプレッドシート「海亀兄弟予約管理」を、スマホ・パソコンで使える管理アプリとして表示するGAS HTMLService版です。

## 現在できること

- 今日・明日・未対応・開催場所未設定・確定売上の確認
- 名前、予約番号、電話番号、プランによる検索
- 日付、ステータス、開催場所による絞り込み
- 予約番号が同じ2行・3行のセット予約を1件にまとめて表示
- 予約詳細と参加者詳細の確認
- 未対応予約の予定ごとの日付・時間変更（予約一覧とGoogleカレンダーを同時更新）
- 重複予約の安全削除（削除済み予約へ退避し、Googleカレンダーも同時削除）
- 予約ステータスの一括更新
- セット内の各予定ごとに開催場所と担当スタッフを更新
- 確定・満席・開催場所・自由メッセージのLINE全文プレビュー
- 明示的な最終確認後だけLINE送信
- `LINE送信履歴`の表示と追記
- `管理アプリ操作履歴`への変更記録
- 他画面で予約が更新された場合の上書き防止

## 安全な構成

- 既存の予約受付GAS `apps-script/umigame-reservation-admin/Code.gs` は変更しません。
- このフォルダのコードは、**新しい管理Webアプリ専用GASプロジェクト**へ配置します。
- Web画面の変更では既存の編集トリガーは発火しません。
- ステータスや開催場所を変更しても、LINEは全文プレビュー画面で最終ボタンを押すまで送信されません。
- LINE送信成功履歴を確認し、同じ予約へ同じ確定・満席通知を二重送信しないようにしています。
- 予約削除には予約番号の再入力が必要で、削除前の全行を`削除済み予約`シートへ退避します。
- ブラウザへLINE User IDや`NOTIFY_SECRET`を返しません。

## 導入手順

### 1. 管理画面専用GASを作成

Googleドライブで新しいApps Scriptプロジェクトを作り、分かりやすく「海亀兄弟 予約管理Webアプリ」と命名します。既存の予約受付GASには貼り付けません。

### 2. ファイルを登録

このフォルダと同じ名前で、次の4ファイルをApps Scriptへ登録します。

- `Code.gs`
- `Index.html`
- `Styles.html`
- `App.html`

マニフェストを表示できる場合は`appsscript.json`も同じ内容にします。

### 3. スクリプトプロパティを設定

Apps Scriptの「プロジェクトの設定」→「スクリプト プロパティ」へ登録します。

| 名前 | 値 | 必須 |
|---|---|---|
| `SPREADSHEET_ID` | `1bPYur4Dfg3LxTCIiYzZvyZRT8bgLoYizG1B6LIkETKk` | 推奨（未設定時もこのIDを使用） |
| `CALENDAR_ID` | `genkidama2439@gmail.com` | 推奨（未設定時もこのIDを使用） |
| `NOTIFY_SECRET` | 既存予約GASと同じLINE通知用シークレット | LINE送信時に必須 |
| `ADMIN_ALLOWED_EMAILS` | 利用を許可するGoogleアカウント。複数はカンマ区切り | 必須 |

秘密情報はこのリポジトリのファイルへ書きません。

### 4. 最初の権限許可

Apps Scriptエディタで`adminGetAppData`を一度実行し、スプレッドシートへのアクセス権限を許可します。続けて `adminAuthorizeCalendarAccess` を実行し、Googleカレンダー権限を許可します。どちらも認可時に予約内容やカレンダー予定を変更しません。

### 5. Webアプリとしてデプロイ

「デプロイ」→「新しいデプロイ」→「ウェブアプリ」を選びます。

- 実行するユーザー：自分
- アクセスできるユーザー：最初は**自分のみ**

発行されたURLをスマホまたはパソコンで開きます。最初は閲覧・検索・詳細確認を行い、テスト予約で更新とLINE送信を確認してから通常運用へ移します。

## 初回確認の順番

1. 予約件数と本日の件数がシートと合う
2. セット予約が1件にまとまり、詳細で2〜3予定に分かれる
3. 専用テスト予約を「未対応」のまま日付・時間変更する
4. 予約一覧とGoogleカレンダーが同じ新日時になり、LINEはまだ届かない
5. そのテスト予約を「確定」にし、LINEプレビューが新日時になっている
6. 「送信しない」で閉じた場合にLINEが届かない
7. 再度プレビューしてテスト送信後、`LINE送信履歴`と`管理アプリ操作履歴`へ記録される
8. 重複させたテスト予約を削除し、予約一覧・Googleカレンダーから消えて`削除済み予約`へ退避される
